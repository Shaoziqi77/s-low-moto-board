# S低L3-3数据处理工具需求文档

## 1 项目背景

根据用户上传的数据表格，按照既定工作流完成数据清洗、关联、衍生字段计算，输出多Sheet汇总数据表；配套提供数据预览、结果图片复制、文件下载、本地缓存等快捷操作功能。

## 2 网页整体设计

页面划分4个文件上传模块，操作流程分两阶段：

1. 仅上传前2个文件：可执行「底表处理」；
2. 上传全部4份文件：可执行完整「数据处理」。

处理完成后配套功能：

- 多Sheet切换预览窗口；
- 指定Sheet数据导出为图片，支持一键复制至剪贴板；
- 成品Excel文件下载。

## 3 业务工作流

整体输出统一工作表文件，根据上传文件与处理规则自动生成多张独立Sheet。

### 3.1 数据上传模块（共4组上传入口）

#### 3.1.1 基础班级数据

1. 上传提示文案：从「可到转介绍班级看板」按「学期ID」筛选查询后下载上传
   看板地址：<https://data.corp.hetao101.com/dc-report-fe/project/6/portal/12/dashboard/250>
2. 上传后生成Sheet：`底表不动（班级）`
   - 保留字段并排序：班级ID、课导ID、课导花名、学期ID、课导组名、课导组长名称、学期名称、班级名称、开课日
   - 剔除原始文件所有多余字段
   - 衍生字段规则：开课日 = 班级名称前两个字符（示例：周四/周五/周六）
3. 页面交互：提取全部班级ID展示，配套「复制」按钮一键复制批量班级ID
   数据示例：
   416988
   416992
   411236

#### 3.1.2 基础用户数据

1. 上传提示文案：从「可到转介绍用户看板」按「班级ID」筛选查询后下载上传
   看板地址：<https://data.corp.hetao101.com/dc-report-fe/project/6/portal/12/dashboard/253>
2. 上传后生成Sheet：`底表不动（用户）`

- 保留字段并排序：用户ID、孩子姓名、学期ID、班级ID、微信名称、开课日
- 剔除原始文件所有多余字段
- 衍生字段规则：开课日通过班级ID，VLOOKUP关联`底表不动（班级）`获取

#### 3.1.3 活动班级数据

1. 上传提示文案：从「MOT-S低超级马里奥活动」按「学期ID」选择「班级维度」查询下载上传
   看板地址：<https://data.corp.hetao101.com/dc-report-fe/project/6/portal/12/dashboard/4081>
2. 上传后生成Sheet：`班级信息`

- 保留原始全部字段，新增衍生字段：开课日
- 衍生字段规则：开课日通过班级ID，VLOOKUP关联`底表不动（班级）`获取

#### 3.1.4 活动用户数据

1. 上传提示文案：从「MOT-S低超级马里奥活动」按「学期ID」选择「用户明细」查询下载上传
   看板地址：<https://data.corp.hetao101.com/dc-report-fe/project/6/portal/12/dashboard/4081>
2. 上传后生成Sheet：`用户明细`

- 字段处理：删除「用户真实姓名」「微信昵称」；隐藏「学期ID」字段
- 空白填充：「好友体验UV」空白单元格统一填充0
- 新增3个衍生字段：孩子姓名、开课日、是否分享、核桃币
  - 孩子姓名：通过用户ID VLOOKUP关联`底表不动（用户）`获取
  - 开课日：通过班级ID VLOOKUP关联`底表不动（班级）`获取
  - 是否分享：IF函数判断，末次分享时间为空则赋值0，有值则赋值1
  - 核桃币：分层IF逻辑判断，规则如下：&#x20;
    - 1\. 若【好友体验UV】≥10 → 核桃币 = 4000核桃币
    - 2\. 若【好友体验UV】≥3 且 好友体验UV＜10 → 核桃币 = 200核桃币
    - 3\. 若【好友体验UV】＜3 且 【是否分享】=1 → 核桃币 = 50核桃币
    - 4\. 其余不满足以上条件的用户，核桃币默认赋值为空
- 排序规则：按「核桃币」对应的核桃币数量降序排列

### 3.2 完整数据处理（4份文件全部上传后执行）

#### 3.2.1 获奖名单

- 新建Sheet：`获奖名单`
- 数据来源：`用户明细`中「奖项」≠「未获奖」的所有行（有核桃币获奖记录）
- 保留字段：用户ID、班级ID、老师ID、老师花名、新用户好友体验UV、奖项、孩子姓名、开课日、是否分享、核桃币

#### 3.2.2 完课已体验用户

- 新建Sheet：`完课已体验用户`
- 数据来源：`用户明细`中满足双重条件：完课时间不为空 **且** 末次体验时间不为空

#### 3.2.3 完课未体验用户

- 新建Sheet：`完课未体验用户`
- 数据来源：`用户明细`中满足双重条件：完课时间不为空 **且** 末次体验时间为空

#### 3.2.4 周x看板

1. Sheet 命名规则（根据用户在筛选器中选择的「开课日」自动命名）
   | 选择的开课日 | 自动生成的Sheet名称 |
   | ------ | ----------- |
   | 单个（如 周四） | 周四看板 |
   | 多个（如 周四、周五） | 周四&周五看板 |
   | 多个（如 周四、周五、周六） | 周四&周五&周六看板 |
   | 未勾选任何或全部勾选后清空 | 全部看板 |
2. 开课日筛选器（看板页内交互）
   - 位置：看板双行标题下方、透视表上方，横向排列的复选框筛选条
   - 选项来源：`班级信息` 中「开课日」列去重后按周一至周日顺序展示，如 周四、周五、周六
   - 辅助按钮：提供「全选」「清空」快捷按钮
   - 交互：勾选/取消勾选后，立即按所选开课日过滤对应行重新生成透视表，并自动更新 tab 名称与 A2 标题（参见第 3 条透视表规则）
3. 在数据预览面板【周x看板】Sheet标签右侧保留「✎ 编辑」按钮
   - 点击后支持用户手动修改当前看板 Sheet 名称，覆盖筛选器自动生成的默认名称
   - 修改后同步更新 Excel 内该 Sheet 工作表名、预览面板标题与导出图片文件名
4. 数据来源：基于`班级信息`，按筛选后的行生成数据透视表（未勾选筛选时默认聚合所有开课日）
   - A1 单元格固定文本：开课日
   - A2 单元格：自动填充上述筛选规则对应的看板名称（或手动编辑后的名称）
   - A3 起为透视表数据区域
     - 行：组长、老师花名
     - 列：值
     - 值（均为求和项，百分比项需按分母计算后再显示）：在班人数、完课人数、完课率、触达人数、完课触达率、进入活动人数、完课进入率、体验作品人数、完课体验率、分享作品人数
     - 末尾增加「总计」汇总行（按数值列求和后重算各百分比）
5. 单元格样式：「完课进入率」「完课体验率」配置红黄绿三色条件色阶（按当前筛选数据的**列极值动态分段**，越高越绿）
   - 先对当前筛选范围内的数据行，计算该列的最小值 min 与最大值 max
   - 中间阈值 mid = (min + max) / 2；upperMid = (mid + max) / 2
   - 绿色：≥ upperMid（高段）
   - 黄色：[mid, upperMid)（中段）
   - 红色：< mid（低段）

### 3.3 数据预览与文件下载

#### 3.3.1 底表下载（仅需上传前2份基础数据）

1. 操作按钮：【底表生成】

- 前置条件：上传「基础班级数据」+「基础用户数据」
- 配套多Sheet切换预览窗口

1. 下载按钮：【文件下载】

- 文件命名规则：提取`底表不动（班级）`内「学期名称」字段缩写生成
- 示例：学期名称=2026-春季图形化编程第3期-1周2课专用（乐高测试）→ 文件名：`S低思维-26春3期底表`

#### 3.3.2 完整数据下载（4份文件全部上传）

1. 操作按钮：【文件生成】

- 前置条件：4个上传模块全部上传完成
- 配套多Sheet切换预览窗口

1. 图片复制功能

- 将`获奖名单`全表导出为图片，支持一键复制剪贴板
- 将`周x看板`全表导出为图片，支持一键复制剪贴板

1. 下载按钮：【文件下载】

- 文件命名规则：提取`底表不动（班级）`内「学期名称」字段缩写生成
- 示例：学期名称=2026-春季图形化编程第3期-1周2课专用（乐高测试）→ 文件名：`S低思维-26春3期`

### 3.4 本地数据暂存

- 功能：本地缓存用户已上传文件数据
- 交互逻辑：工具打开时弹窗提示「是否恢复上次缓存数据」，避免重复上传文件
